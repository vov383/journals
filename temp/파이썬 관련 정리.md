# 컴퓨터 일반 파이썬 예제 분석

최근 국가직 및 지방직 9급 전산직 시험의 '컴퓨터 일반' 과목에서 프로그래밍 언어론의 비중이 유지되고 있으며, 
기존 C와 Java 중심에서 Python의 문법적 특성을 묻는 문제가 지속적으로 출제되는 경향임. 
특히 단순 문법보다는 파이썬 고유의 **자료구조(List, Set, Dictionary) 처리 방식**과 
**슬라이싱**, **재귀함수** 로직을 묻는 유형이 주를 이룸.

주요 기출 패턴을 5가지 핵심 유형으로 분류하여 정리함.

## 1. 문자열 및 리스트 슬라이싱 (Slicing)

파이썬의 가장 큰 특징인 슬라이싱 기능을 활용하여 인덱스 계산 능력을 평가함. 음수 인덱스(`-1`)와 스텝(`step`) 처리가 핵심임.

- **출제 포인트**
    
    - `[start:end:step]` 구조의 이해
        
    - `end` 인덱스는 포함되지 않음(Exclusive)
        
    - 음수 인덱스를 통한 역순 접근
        

###### 예시코드 제목: 슬라이싱 및 음수 인덱스 예제


```python
str_data = "GwanganBridge"
list_data = [10, 20, 30, 40, 50, 60]

# 1. 문자열 역순 출력
print(str_data[::-1]) 
# 결과: egdirBnagnawG

# 2. 리스트 부분 추출 (인덱스 1부터 4직전까지, 2칸씩)
print(list_data[1:4:2])
# 결과: [20, 40] (인덱스 1(20), 인덱스 3(40) 선택됨)

# 3. 음수 인덱스 슬라이싱
print(list_data[-3:])
# 결과: [40, 50, 60] (뒤에서 3번째부터 끝까지)
```

## 2. 제어문과 Range 함수의 활용

`for`문과 `range()` 함수를 조합한 반복 로직 계산 문제임. 
C언어와 달리 `range(start, stop)`에서 `stop` 값이 포함되지 않는다는 점을 명확히 알고 있어야 함.

- **출제 포인트**
    
    - `range(n)`: 0부터 n-1까지
        
    - `print(..., end=' ')`: 줄바꿈 없이 출력하는 옵션
        
    - 중첩 루프에서의 실행 횟수 계산
        

###### 예시코드 제목: Range 함수와 출력 형식을 묻는 예제

Python

```Python
sum = 0
# 1부터 10 '직전'까지 2씩 증가 (1, 3, 5, 7, 9)
for i in range(1, 10, 2):
    sum += i

print("Sum:", sum)
# 결과: Sum: 25

# 별표 출력 로직 (end 파라미터 주의)
for i in range(3):
    print("*", end='')
# 결과: *** (줄바꿈 없이 가로로 출력)
```

## 3. 람다(Lambda) 함수와 Map/Filter

함수형 프로그래밍의 특징인 람다 표현식과 고차 함수(`map`, `filter`)의 동작 결과를 예측하는 문제임. 
코드가 간결하여 실수하기 쉬움.

- **출제 포인트**
    
    - `lambda 매개변수: 반환값` 형식 이해
        
    - `map`은 모든 요소에 함수 적용, `filter`는 조건이 참인 요소만 추출
        
    - 최신 버전에서는 `list()`로 감싸지 않으면 객체 주소가 반환되나, 문제에서는 주로 리스트 변환된 결과를 묻거나 값 자체를 논리적으로 물음.
        

###### 예시코드 제목: 람다와 Filter 활용 예제

```Python
numbers = [1, 2, 3, 4, 5, 6]

# 짝수만 골라내어 제곱하는 로직
# 1. filter로 짝수 추출: [2, 4, 6]
# 2. map으로 제곱 적용: [4, 16, 36]
# 3. list로 변환하여 출력
result = list(map(lambda x: x**2, filter(lambda x: x % 2 == 0, numbers)))

print(result)
# 결과: [4, 16, 36]
```

## 4. 가변(Mutable) vs 불변(Immutable) 객체와 함수 인자 전달

C#이나 Java 개발자가 파이썬 문제에서 가장 많이 혼동하는 영역임. 
파이썬은 "Call by Object Reference" 방식을 사용하므로, 
전달된 객체가 변경 가능한지(List, Dict) 불가능한지(Int, Str, Tuple)에 따라 부작용(Side Effect)이 다름.

- **출제 포인트**
    
    - 리스트를 함수 인자로 넘겨 내부에서 `append` 등을 했을 때 원본이 바뀌는지 여부 (바뀜)
        
    - 정수(Int) 값을 넘겨 내부에서 재할당했을 때 원본이 바뀌는지 여부 (안 바뀜)
        

###### 예시코드 제목: 리스트(Mutable)와 정수(Immutable)의 참조 전달 차이


```Python
def modify_data(n, l):
    n = n + 10      # 정수는 불변 객체. 새로운 20 객체가 생성되고 로컬변수 n이 가리킴
    l.append(4)     # 리스트는 가변 객체. 원본 리스트 주소에 접근하여 4를 추가함

a = 10
b = [1, 2, 3]

modify_data(a, b)

print(a, b)
# 결과: 10 [1, 2, 3, 4]
# a는 변경되지 않았으나, b는 변경되었음이 핵심
```

## 5. 재귀 함수(Recursion) 트레이싱

정보처리기사나 공무원 시험 공통적으로 등장하는 고난도 유형. 
팩토리얼, 피보나치 등을 파이썬 문법으로 구현하고 실행 순서나 최종 리턴값을 물음.

- **출제 포인트**
    
    - 기저 조건(Base Case) 확인
        
    - 재귀 호출의 스택 구조 추적
        
    - `or`, `and` 연산자와 결합된 리턴 로직
        

###### 예시코드 제목: 재귀 호출 실행 흐름 예제

```Python
def func(n):
    if n <= 1:
        return 1
    else:
        return n + func(n - 2)

# 실행 흐름:
# func(5) 호출 -> 5 + func(3)
# func(3) 호출 -> 3 + func(1)
# func(1) 호출 -> 1 반환
# 역순 계산: 1 -> 3+1=4 -> 5+4=9
print(func(5))
# 결과: 9
```

## 6. 파이썬 고유 연산자 및 자료구조 메소드

C 계열 언어에는 없는 
파이썬만의 연산자나 메소드 기능을 묻는 단답형 코드 문제.

- **주요 내용**
    
    - `**` (거듭제곱), `//` (몫 연산자)
        
    - `set` 자료형의 중복 제거 특성
        
    - `dictionary`의 `get()` 메소드 등
        

###### 예시코드 제목: 연산자 및 Set 활용 예제

```Python
# 1. 연산자 우선순위 및 기능
print(2 ** 3)   # 8 (거듭제곱)
print(7 // 2)   # 3 (몫)

# 2. Set을 이용한 중복 제거
data = [1, 1, 2, 2, 3, 3, 3]
unique_data = set(data)
print(unique_data)
# 결과: {1, 2, 3} (순서는 보장되지 않으나 요소는 유니크함)
```

1. **슬라이싱**: `[start:end:step]` 문법에서 `end`가 포함되지 않음과 음수 인덱스 처리가 빈출됨.
    
2. **반복문**: `range` 함수의 범위와 `print` 함수의 `end` 옵션 처리를 통한 출력 형식 확인 필요.
    
3. **함수형 기능**: `lambda`, `map`, `filter`를 조합한 데이터 처리 결과 예측 능력 요구.
    
4. **객체 참조**: 가변(List) 객체와 불변(Int, Str) 객체의 함수 전달 시 원본 데이터 변경 여부(Side Effect) 구분 필수.
    
5. **기타**: 재귀 함수의 스택 트레이싱과 파이썬 전용 연산자(`**`, `//`)는 점수 확보를 위해 반드시 숙지해야 함.
